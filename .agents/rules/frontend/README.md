# Frontend Rules

1. Recommended tech stack is Next.js 16 App Router with TypeScript and Tailwind CSS.
2. Use const functional components only and keep each component at 80 lines or less.
3. Use Tailwind CSS for all styling; avoid third-party UI component libraries.
4. Enforce accessibility and semantic HTML across all public UI surfaces.
5. Maintain bilingual content parity for public-facing pages (English and Spanish must both be complete and consistent).
6. Use Tailwind CSS via CDN and avoid custom CSS unless strictly necessary.
7. Implement all form validation client-side with clear, user-facing error states.
8. Keep a shared header, navigation, and footer across landing and form pages.
9. Include required Schema.org markup and semantic HTML for SEO and accessibility.
